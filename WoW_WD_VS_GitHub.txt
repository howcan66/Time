--------------------------------------
ALWAYS RUN:

  - Pushing your changes
  - Switching branches

HANDLES CONFLICTS:
  - If someone else pushed changes

STEP 7.2: Resolve Merge Conflicts (If Occur)
---------------------------------------------
CONFLICT INDICATORS IN FILE:
  <<<<<<< HEAD
  Your changes
  =======
  Their changes
  >>>>>>> origin/main

RESOLVE IN VS CODE:
  1. VS Code highlights conflicts
  2. Choose: "Accept Current Change" or "Accept Incoming Change" or "Accept Both"
  3. Edit manually if needed
  4. Remove conflict markers
  5. Save file
  6. Stage resolved file
  7. Commit merge

  git add resolved-file.html
  git commit -m "MERGE: Resolve conflict in kanban.html"

SECTION 8: TROUBLESHOOTING COMMON ISSUES
================================================================================

ISSUE 8.1: Cannot Push - "Rejected" Error
------------------------------------------
CAUSE: Remote has changes you don't have locally

SOLUTION:
  git pull origin main
  # Resolve any conflicts
  git push origin main


ISSUE 8.2: Live Server Not Working
-----------------------------------
SYMPTOMS: Browser doesn't open or doesn't auto-refresh

SOLUTION:
  1. Check extension installed: Extensions ‚Üí Search "Live Server"
  2. Right-click HTML file (not in editor, in File Explorer)
  3. Check firewall/antivirus not blocking port 5500
  4. Try different port: Settings ‚Üí Live Server ‚Üí Port ‚Üí 5501


ISSUE 8.3: Git Not Recognized in VS Code
-----------------------------------------
SYMPTOMS: Source Control shows "No repository found"

SOLUTION:
  1. Verify git installed: Terminal ‚Üí git --version
  2. Check .git folder exists in project root
  3. Reload VS Code: Ctrl+Shift+P ‚Üí "Developer: Reload Window"
  4. Check git.path in settings


ISSUE 8.4: Changes Not Showing in Browser
------------------------------------------
CAUSE: Browser cache

SOLUTION:
  - Hard refresh: Ctrl+Shift+R (Chrome/Edge)
  - Or: Ctrl+F5
  - Or: Clear cache in DevTools (F12 ‚Üí Network ‚Üí Disable cache)
  - Or: Restart Live Server


ISSUE 8.5: Merge Conflict Too Complex
--------------------------------------
SOLUTION (Nuclear Option):
  1. Backup your changes (copy files)
  2. Abort merge: git merge --abort
  3. Discard local changes: git reset --hard origin/main
  4. Manually re-apply your changes from backup
  5. Commit fresh

BETTER SOLUTION:
  - Commit frequently (smaller conflicts)
  - Pull before making major changes
  - Communicate with team about file edits


ISSUE 8.6: Accidentally Committed Wrong Files
----------------------------------------------
UNDO LAST COMMIT (Keep Changes):
  git reset --soft HEAD~1

UNDO LAST COMMIT (Discard Changes):
  git reset --hard HEAD~1  # DESTRUCTIVE - USE CAREFULLY

REMOVE FILE FROM STAGING:
  git restore --staged filename.html

SECTION 12: ADVANCED WORKFLOWS
================================================================================

WORKING WITH BRANCHES:
  # Create feature branch
  git checkout -b feature/new-feature
  
  # Work on feature (commits)
  git commit -m "FEAT: Part 1 of new feature"
  
  # Push branch to GitHub
  git push origin feature/new-feature
  
  # Switch back to main
  git checkout main
  
  # Merge feature to main
  git merge feature/new-feature
  
  # Delete branch (after merge)
  git branch -d feature/new-feature

STASHING WORK IN PROGRESS:
  # Save work without committing
  git stash
  
  # Switch to main, pull, etc.
  git checkout main
  git pull
  
  # Restore stashed work
  git stash pop

VIEW COMMIT HISTORY:
  # VS Code: Timeline view (bottom of File Explorer)
  # GitLens: Hover over line ‚Üí See who changed it
  # Terminal: git log --oneline --graph

REVERT TO PREVIOUS VERSION:
  # Restore single file from last commit
  git checkout HEAD -- filename.html
  
  # Restore from specific commit
  git checkout <commit-hash> -- filename.html

COMPARE CHANGES:
  # VS Code: Click modified file in Source Control
  # Terminal: git diff filename.html
  # GitHub: Compare commits online
