PROVEN PATTERNS
================================================================================

PATTERN 1: Ask Before Executing
  When: User provides ambiguous instructions
  How: Analyze options, ask for confirmation, wait for response
  Success Rate: 100% (prevents wrong implementation)

PATTERN 2: Template-Driven Workflow
  When: Tasks pass through multiple stages
  How: Create template for each stage with required fields
  Success Rate: High (consistency and faster processing)

PATTERN 3: Beginner-Focused Documentation
  When: Users have basic knowledge
  How: Break topics into small chunks, include practical examples
  Success Rate: High (users feel confident and don't need clarification)

PATTERN 4: Decision-First Development
  When: Task has multiple possible approaches
  How: Present options first, wait for direction, then implement
  Success Rate: High (matches user's actual needs)
