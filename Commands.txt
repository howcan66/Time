================================================================================
CORE WORKFLOW COMMANDS
================================================================================

P2G = Publish to Git / Push to GitHub
- Stage, commit, and push changes to GitHub
- Example: "P2G" ‚Üí git add, commit, push

PA = Please Add / Please Analyze
- Request to add feature or analyze task/idea
- Provides detailed report on requirements, approach, dependencies
- Example: "PA Feature 12" ‚Üí Add new feature
- Example: "AI TD15 PA" ‚Üí Analyze Interactive Calendar task

PE = Please Execute / Proceed Execute
- Execute after analysis approval
- PE: Execute single item
- PEX: Execute everything pending
- Example: "PE HJ-ID1D4" ‚Üí Implement design

PWF = Propose Way Forward
- AI proposes next steps or solution approach
- Always included in responses when requested
- Example: "Always Propose Way FWD"

EX = Execute
- Direct execution without prior analysis (use cautiously)
- Example: "EX create backup"

CONFIRM = Approve and Continue
- Confirms proposed action
- Proceeds with implementation

BLOCK = Stop/Reject
- Rejects proposed action
- Requires alternative approach

REVISE = Modify Approach
- Request changes to proposed solution
- Requires new analysis

RETEST = Test Again
- Re-run tests after fixes
- Example: "RETEST AI TD15"

CLOSE = Mark Complete
- Finalizes task as done
- Moves to Completed.txt

================================================================================
DOCUMENTATION & REVIEW COMMANDS
================================================================================

AL = Analyze Log
- Scan Log_all.txt for redundancy/noise
- Propose deletions or condensed summaries
- Example: "AL" ‚Üí Review and clean logs

AD = Analyze Documentation
- Review Requirements.txt, Design.txt, Instructions.txt
- Flag overlaps, outdated sections
- Recommend improvements
- Example: "AD" ‚Üí Documentation audit

UD = Update Documentation
- Update cross-references and status in docs
- Example: "UD after RS4"

LA = Log All
- Log activity to Log_all.txt
- Example: "LA implementation complete"

================================================================================
KANBAN & TASK MANAGEMENT COMMANDS
================================================================================

UKB = Update Kanban Backlog
- Add tasks from Todo.txt to Kanban board
- Example: "UKB" ‚Üí Sync tasks to UI

UKB-Low = Update Kanban - Low Priority
- Add low-priority tasks to Kanban
- Example: "UKB-Low" ‚Üí Add AI TD19, TD21, TD26

PSC = Propose Status Change
- AI suggests status change (requires PM approval)
- Example: "PSC move to Test"

RSC = Request Status Change
- PM (user) approves status change
- Example: "RSC approved ‚Üí Test"

PROPOSED NEW COMMANDS
================================================================================

CMD = Show Commands
- Display this file or command reference
- Example: "CMD" ‚Üí Show all available commands

QS = Quick Status
- Show current project status (what's in progress, pending, done)
- Example: "QS" ‚Üí Status overview

TB = Test Build
- Run/test current implementation
- Example: "TB kanban.html" ‚Üí Open in browser

BK = Backup
- Create backup of specified file(s)
- Example: "BK kanban.html" ‚Üí Create backup

RB = Rollback
- Restore from backup
- Example: "RB kanban.html" ‚Üí Restore previous version

SYNC = Synchronize
- Sync data between files (Todo.txt ‚Üî Kanban)
- Example: "SYNC Todo to Kanban"

EXPORT = Export Data
- Export from Kanban or other tool
- Example: "EXPORT Kanban"

IMPORT = Import Data
- Import tasks/data from file
- Example: "IMPORT tasks.txt"

CLEAN = Clean/Optimize
- Remove unused code, optimize files
- Example: "CLEAN kanban.html"

SPLIT = Split Task
- Break large task into smaller subtasks
- Example: "SPLIT AI TD15 into 3 tasks"

MERGE = Merge Tasks
- Combine related tasks
- Example: "MERGE AI TD12 and TD13"

ASSIGN = Assign to Project
- Assign task to specific Idea/Project
- Example: "ASSIGN AI TD15 to proj-2"

PRIORITY = Set Priority
- Change task priority
- Example: "PRIORITY AI TD15 = 1"

ARCHIVE = Archive Item
- Move to archive (out of active view)
- Example: "ARCHIVE old logs"

SEARCH = Search Content
- Search across files
- Example: "SEARCH drag-drop"

USAGE EXAMPLES
================================================================================

Single Command:
"PA" ‚Üí Analyze current context
"AL" ‚Üí Analyze logs
"UKB" ‚Üí Update Kanban backlog

Combined Commands:
"PA AI TD15" ‚Üí Analyze task AI TD15
"PE HJ-ID4" ‚Üí Execute HJ-ID4 implementation
"AL then AD" ‚Üí Analyze logs, then documentation
"UKB then QS" ‚Üí Update backlog, show status

Chained Workflow:
"PA ‚Üí (analysis provided) ‚Üí PE ‚Üí TEST ‚Üí CLOSE"

================================================================================
BEST PRACTICES
================================================================================

1. Use PA before major changes
2. Use AL/AD regularly to keep docs clean
3. UKB when adding many tasks
4. Always wait for CONFIRM before executing major changes
5. Use AQ when uncertain about requirements
6. Log major milestones with LA
