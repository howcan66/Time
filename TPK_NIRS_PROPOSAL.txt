================================================================================
PROPOSED: TPK-NIRS FORMAT
================================================================================

Structure: TPK-[VERSION]-[CATEGORY]-[PRIORITY]-[SEQUENCE]

Example: TPK-v1-SEC-01-001
  TPK = Time Project Kanban (prefix)
  v1 = Version 1
  SEC = Security (category)
  01 = Priority 1 (Critical)
  001 = Sequential number (001, 002, 003...)

Full Format Breakdown:

1. PREFIX: TPK
   Meaning: Time Project Kanban
   Purpose: Identify project
   Fixed: Yes

2. VERSION: v1, v2, v3...
   Meaning: Release version
   Examples: v1 (current), v2 (next major), v3 (future)
   Current: v1

3. CATEGORY: 3-4 letter code
   Meaning: Feature type/category
   Examples: SEC, ADM, EXP, COM, RES, ORG, TSK, DOC, CORE, TEST, BUG, FIX
   Purpose: Quick identification of feature type

4. PRIORITY: 01-99 (numeric)
   Meaning: Priority level (01=Critical, 02=High, 03=Medium, 04=Low)
   Mapped to:
     01 = CRITICAL (security, auth, core functionality)
     02 = HIGH (major features, user-facing)
     03 = MEDIUM (enhancements, improvements)
     04 = LOW (nice-to-have, minor fixes)

5. SEQUENCE: 001-999 (numeric)
   Meaning: Sequential number within category-priority
   Examples: 001 (first critical security), 002 (second critical security)
   Purpose: Unique identifier within classification

Each TPK-NIRS ID requires a Requirement Specification document with:

   - Feature Name: Password Visibility Toggle
   - Category: Security
   - Priority: Critical (01)
   - Status: COMPLETED / IN_PROGRESS / PENDING / ON_HOLD
   - Version: v1
   - Created: 2026-01-29
   - Owner: AI
   - Deadline: 2026-01-30

2. DESCRIPTION SECTION
   - Executive Summary (1-2 sentences)
   - Detailed Description (paragraph)
   - User Stories (format: "As a [role], I want [action], so [benefit]")
   - Acceptance Criteria (checklist)

3. TECHNICAL SECTION
   - Technical Requirements
   - Architecture/Design
   - Dependencies
   - Implementation Notes
   - Test Coverage

4. STATUS TRACKING SECTION
   - Current Status
   - Progress Percentage
   - Blockers
   - Next Steps
   - Commit Reference (if implemented)

5. DOCUMENTATION SECTION
   - Related Files
   - Test Cases
   - Design Documents
   - References to other specs
